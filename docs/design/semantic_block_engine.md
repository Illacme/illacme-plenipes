# 语义块引擎与状态机解析规约

## 1. 设计目标
为了实现“手术刀级”的增量同步，引擎必须具备将 Markdown 文档拆解为原子语义块的能力。本设计旨在确保文档的样式（如嵌套 Callout）在切片后仍能 100% 还原，并为每个块生成唯一的“数字指纹”。

---

## 2. 解析器核心逻辑 (State Machine)

### 2.1 块类型定义 (Block Types)
*   **Header**: 标题块（H1-H6）。标题的变动会触发元数据更新。
*   **Paragraph**: 常规正文块。
*   **Callout**: 提醒/警告块。采用深度优先扫描，确保 `> [!TIP]` 等语法及其内容被完整捕获。
*   **Spacer**: 块间空行。主要用于维持文档物理结构的完整性，通常不消耗 AI 算力（直传）。

### 2.2 解析算法：行级状态机
`MarkdownBlockParser` 采用单次迭代的状态机模型：
1.  **捕获态**：识别块的起始标识（如 `>` 或 `#`）。
2.  **持续态**：在未遇到块结束标识（如连续空行或类型切换）前，持续向当前块追加内容。
3.  **闭合态**：计算当前块内容的 MD5，并标记其类型。

---

## 3. 指纹生成协议 (Fingerprinting)
指纹（Fingerprint）是块级复用的唯一索引，计算公式如下：
$$Fingerprint = MD5(BlockType + Content.strip())$$

**注意**：指纹计算排除了块首尾的冗余空白符，以增强对格式微调的容错性。

---

## 4. 增量组装逻辑 (Assembly)
在同步阶段，引擎将 $N$ 个块按照原始解析顺序进行串联。影子存储中的译文碎片被无损地填充回对应的位置，确保最终生成的 Markdown 文件具备完美的排版一致性。
