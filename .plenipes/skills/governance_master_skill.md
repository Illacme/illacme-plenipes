# 🛡️ Illacme-plenipes 全自动治理技能 (Governance Master Skill)

## 📋 技能描述
本技能是 Illacme-plenipes 引擎的“灵魂约束”，确保 AI 智能体在任何开发、维护或重构过程中，始终保持工业级的严谨性与架构纯净度。

## ⚡ 触发条件
- 任何代码修改任务开始前
- 任何代码提交 (git commit) 发起前
- 用户明确要求进行“审计”或“检查”时

## 🛠️ 执行流程 (SOP)

### 第一阶段：认知同步
1. **读取教训**：在动手前，必须物理读取 `.plenipes/evolution_records.md`，确认是否有与当前任务相关的“踩坑记录”。
2. **状态检查**：确认当前是否处于“架构复健期”（参考 TDR 节律）。

### 第二阶段：实施约束 (Hard Constraints)
1. **零占位符协议**：禁止生成任何包含 `# ... skip` 或 `# ... existing code` 的代码片段。
2. **注释主权**：严禁在未获授权的情况下删除、精简原有的中文注释和 Docstring。
3. **原子提交**：每次提交必须包含对应的测试用例或历史归档。

### 第三阶段：自动化门禁 (Physical Audit)
1. 在任务结束前，必须主动运行 `python3 tests/governance_audit.py`。
2. **熔断机制**：若审计报告中出现 [FAIL] 项，禁止向用户报告任务完成，必须先行修复直至 100% 通过。

## 🧬 自进化指令
如果我在执行过程中发现了新的 Bug 模式或更优的治理逻辑，我必须建议用户更新本 Skill 文件，实现治理能力的持续进化。
