# 规则自进化 (Meta-Evolution) 设计方案

## 架构痛点
当前的智能体进化模式处于“事后惩罚学习”阶段。即使我们在 `tests/autonomous_simulation.py` 中写满了防线，智能体也只能在代码错误、触碰机制时依靠 Traceback 报错来强行修正行为。这会带来无尽的重复算力浪费，这并不是真正意义上的“智能衍生”。

## 实施路径
在本次 Iteration 中，打破 Agent 仅能修改执行层代码（src/）和数据域的壁垒，向最高权力发起授权：
**直接修改项目规则集**（`.plenipes/rules.md`）。

- **修宪行动**：加入章节 `IX. 规则元进化 (Meta-Evolution)`。
- **强制指令**：明确规定未来的智能体，无论在处理 Git 冲突、填补规则漏洞，还是开辟了新约束时，必须主动使用编辑工具回到 `rules.md` 沉淀经验，而非依赖测试脚手架“撞南墙”。
